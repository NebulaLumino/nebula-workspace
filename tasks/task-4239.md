## Overview
Build an AI-powered web app that generates Architecture Decision Records (ADRs) from a description of a technical decision. Users describe a proposed architecture change (e.g., "migrate from REST to GraphQL", "adopt Kubernetes", "switch from Postgres to DynamoDB"), and the AI generates a structured ADR document covering context, decision, consequences, alternatives considered, and references.

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- OpenAI SDK (compatible with DeepSeek)

## Features
- Natural language architecture decision → ADR document
- ADR format selection: MADR (Markdown Any Decision Record), Nat Pryce's format, Michael Nygard's format
- Context section: business drivers, technical constraints, team considerations
- Decision section with clear rationale
- Consequences: pros/cons, trade-offs, risks
- Alternatives considered with rejection reasons
- Stakeholder approver and decision date metadata
- Effort/cost assessment (dev time, operational cost, learning curve)
- "Related ADRs" cross-linking
- ADR status tracking: proposed, accepted, deprecated, superseded
- ADR numbering (auto-increment based on existing ADRs)
- Team discussion prompts and open questions section
- Export as Markdown, JSON, or Confluence-compatible format
