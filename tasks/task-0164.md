## Research Findings — AI Code Execution Environments 2026

### The Big Three

**OpenAI Codex**
- Cloud-based software engineering agent (macOS app launched Feb 2026, CLI, API)
- Deeply integrated: executes terminal commands, runs tests, proposes PRs
- Multi-agent workflows for issue triage, CI/CD automation
- Operates in isolated containers (ephemeral)
- Available via Cursor IDE, ChatGPT code interpreter, OpenAI API

**Anthropic Claude**
- Sandboxed code execution for Bash + file operations
- Programmatic Tool Calling: Claude writes/runs Python scripts to orchestrate tools
- Tool Search Tool: discovers and uses tools dynamically
- Claude Code: terminal CLI, VS Code extension, desktop app
- Supports MCP (Model Context Protocol) for external data connections
- Spawns sub-agents in parallel with own context windows

**E2B Code Interpreter**
- Open-source SDK for AI agent code execution infrastructure
- Firecracker microVMs → hardware-level isolation (stronger than Docker)
- Cold start: ~150ms
- Ephemeral sandboxes (stateless), Jupyter server inside
- Python + JS/TS SDKs, integrates with LangChain, AutoGen, CrewAI
- Streaming: stdout, stderr, charts

### Sandbox Isolation Technologies

| Tech | Used By | Security Level |
|------|---------|---------------|
| Firecracker microVM | E2B, Fly.io Sprites, Daytona | ⭐⭐⭐ Hypervisor (hardware) |
| gVisor | Modal | ⭐⭐ User-space kernel |
| Kata Containers | Northflank | ⭐⭐⭐ VM-level |
| Docker (standard) | Most others | ⭐ Shared host kernel (weakest) |

### Cold Start Latency (2026)

- Blaxel: ~25ms
- Daytona: ~90ms
- E2B: ~150ms
- Modal: <1 second
- Fly.io Sprites: 1–12 seconds

### Session Persistence
- Most are ephemeral (created → used → destroyed)
- Fly.io Sprites: persistent VMs with instant checkpoint/restore
- Trend: agents increasingly need stateful sessions

### GPU Support
- Modal: strongest GPU support (serverless Python)
- E2B: CPU-focused currently
- Important for ML-heavy tasks (model inference, data science)

### Key Architectural Patterns

**Stateless Execution:** Agent runs code in sandbox → returns result → sandbox is destroyed. No state persistence. Good for isolation, bad for multi-step workflows.

**Tool Orchestration:** Claude's Programmatic Tool Calling pattern — agent writes code that calls multiple tools in parallel. More efficient than sequential tool calls.

**MCP (Model Context Protocol):** Emerging open standard for connecting AI agents to external data sources. Similar to LangChain tools but more standardized.
