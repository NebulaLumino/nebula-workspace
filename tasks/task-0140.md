**Status:** ✅ Completed

## Research Summary

### Why Edge AI?
Run AI locally on device instead of cloud — privacy, latency, offline, cost benefits.

### On-Device LLMs

**Apple Intelligence** (on-device)
- 3B+ parameter models on A17/M-chips
- Private Cloud Compute: splits processing between device and private cloud

**Google Gemini Nano**
- Built into Android devices
- Gboard smart reply, Recorder summaries

**Mistral 7B / Llama 3 8B**
- Small enough to run on laptops, even CPUs
- Ollama: run any model locally with one command
-llama.cpp: quantize models to 4-bit, run on MacBooks

### Model Compression Techniques
- **Quantization**: FP16 → INT8 → INT4 → even 1-bit (BitNet)
- **Pruning**: remove low-importance weights
- **Knowledge distillation**: train small model to mimic large model
- **ONNX Runtime**: cross-platform optimized inference

### Tools & Frameworks
- **Ollama**: `ollama run llama3` — locally, in one command
- **llama.cpp**: C++ inference, 4-bit quantized models
- **MLX** (Apple Silicon): optimized for Mac GPU/Neural Engine
- **TensorFlow Lite**: mobile deployment

### Project Ideas
- Local privacy-focused AI writing assistant (Mac app with MLX)
- On-device personal AI that never sends data to the cloud
- Edge AI image generation on laptop (SDXL + llama.cpp)
