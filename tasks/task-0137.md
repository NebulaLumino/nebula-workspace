**Status:** ✅ Completed

**Type:** Research Task

## Research Summary

### Why AI Testing Matters
Traditional software tests can't verify whether AI outputs are correct, safe, or high-quality. New tooling is needed.

### Key Tools

**RAGAS** (ragasdocs.com)
- Evaluation framework for RAG pipelines
- Metrics: faithfulness, answer relevance, context precision
- Works with LangChain, LlamaIndex

**Trulens**
- OSS evaluation for LLM apps
- Trace every LLM call with inputs/outputs
- Feedback functions for groundedness, relevance, safety

**AgentOps**
- Evaluate AI agents: task completion, tool usage, cost
- Session replay for debugging agent behavior

**Phoenix** (Arize)
- LLM observability: latency, token usage, drift detection
- Semantic layer for evaluating QA outputs

**BigCode / SWE-bench**
- Benchmark for code generation (solve real GitHub issues)
- Used to evaluate code LLMs

### Evaluation Metrics
- **Hallucination detection**: Is output grounded in context?
- **Toxicity**: Does output contain harmful content?
- **Faithfulness**: Does answer match the source documents?
- **Task completion**: Did the agent achieve the goal?

### Project Ideas
- LLM evaluation dashboard (Next.js + RAGAS + Phoenix)
- Automated red-teaming framework for AI safety
