## Concept & Vision
A tool for XR developers, UX designers, and spatial computing architects to design intuitive interfaces for AR/VR/MR applications. Generates gaze, gesture, and voice interaction paradigms, spatial layout principles, and accessibility guidelines. Feels like having a spatial computing UX expert design your entire interaction system.

## Specific Features
- Device type selection (Vision Pro/Quest/HoloLens/Magic Leap/custom)
- Interaction modality design (gaze/hand/voice/controller/eye-tracking)
- Spatial UI placement (world-locked/head-locked/body-locked)
- Depth layer design (near/mid/far field interactions)
- Comfort and safety zone mapping
- Gesture vocabulary design
- Voice command taxonomy
- Transitions and spatial animation guidelines
- Accessibility and accommodation options
- Multi-user collaboration patterns

## Input Fields
- Target device and hardware
- Application type (productivity/entertainment/education/training/social)
- Primary interaction modalities
- Target user demographics (age, tech literacy, accessibility needs)
- Environment type (seated/standing/room-scale)
- Multi-user requirements

## Output
Spatial computing interface specification:
- Interaction modality matrix
- Spatial UI layout guide (depth and placement)
- Gesture command library with descriptions
- Voice command taxonomy
- Comfort zone documentation
- Accessibility features checklist
- Multi-user interaction patterns
- Animation and transition guidelines

## Tech Stack
Next.js 16, TypeScript, Tailwind CSS, DeepSeek API, JSON/PNG export.
