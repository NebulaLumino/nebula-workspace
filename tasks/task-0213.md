## Research Findings — Local Embedding Models 2026

### Why Local Embeddings?
- Privacy: data never leaves your machine
- Cost: free after initial download
- Speed: fast inference on local GPU or CPU
- Offline: works without internet

### Top Local Embedding Models

| Model | Provider | Size | Performance | Notes |
|---|---|---|---|---|
| **Nomic Embed Text v1.5** | Nomic | ~275MB | Excellent | Best open-source, CPU-friendly |
| **BGE-M3** | BAAI (FlagEmbedding) | ~1.3GB | SOTA | Multi-lingual, dense + sparse |
| **e5-mistral-7b** | Microsoft | ~14GB | Very high | Best quality, needs GPU |
| **gte-Qwen2-1.5B** | Alibaba | ~3GB | High | Good CPU performance |
| **MiniLM** | Microsoft | ~130MB | Good | Fastest, smallest |
| **OpenAI ada-002** | OpenAI | API | SOTA | Best quality but cloud-only |

### Practical Recommendations

**For Nova's use case (privacy-first RAG):**
1. **Nomic Embed Text v1.5** — best all-around, runs on CPU, well-documented
2. **BGE-M3** — best quality if GPU available
3. **MiniLM** — best if speed is critical

### Ollama Integration
```bash
# Nomic embed text via Ollama
ollama pull nomic-embed-text
ollama embeddings -m nomic-embed-text -p "your text here"

# Or via API
curl http://localhost:11434/api/embeddings -d '{"model":"nomic-embed-text","prompt":"..."}'
```

### Building a Local RAG Pipeline
1. Chunk documents (text → sentence/paragraph chunks)
2. Generate embeddings with local model
3. Store in local vector DB (ChromaDB, Qdrant)
4. Query: embed question → similarity search → LLM answer
