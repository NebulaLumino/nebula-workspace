## Overview
Build an AI-powered web app that generates Terraform (HCL) and Pulumi (TypeScript/Python) infrastructure-as-code from a description of the desired cloud architecture. Users describe their infrastructure needs (e.g., "a VPC with public/private subnets, ECS Fargate cluster, RDS Postgres, ElastiCache Redis, ALB with HTTPS, and IAM roles following least privilege"), and the AI generates complete, production-ready IaC.

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- OpenAI SDK (compatible with DeepSeek)

## Features
- Natural language → Terraform HCL or Pulumi code
- Cloud provider: AWS, GCP, Azure, or multi-cloud
- VPC design: subnets (public/private), NAT gateway, VPC peering, TGW
- Compute: EC2, ECS Fargate, EKS, Lambda, Cloud Run, Azure Container Instances
- Database: RDS, Aurora, DynamoDB, ElastiCache, CosmosDB, Cloud SQL
- Networking: ALB/NLB, Route 53, CloudFront, API Gateway, PrivateLink
- Security: IAM roles/policies, Security Groups, NACLs, KMS, Secrets Manager
- Storage: S3, EFS, FSx, Azure Blob
- Terraform module structure (modules + root module)
- State management and backend config (S3+DynamoDB for Terraform)
- Variable files (tfvars) with sensible defaults
- Cost estimation tags and resource tagging strategy
- drift detection recommendations
- Terraform validate / plan / apply workflow documentation
