## Overview
Build an AI-powered web app that generates production-ready Dockerfiles and Docker Compose files from a description of the application stack. Users describe their application (e.g., "a Node.js + React app with Postgres, Redis, and a Nginx reverse proxy"), and the AI generates optimized multi-stage Dockerfiles, docker-compose.yml with proper networking, health checks, and environment variable management.

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- OpenAI SDK (compatible with DeepSeek)

## Features
- Natural language stack description → Dockerfile + docker-compose.yml
- Language/framework auto-detection: Node.js, Python, Go, Rust, Java, Ruby, .NET
- Multi-stage build optimization (dev vs prod stages)
- Best-practice base image selection (Alpine, slim, distroless)
- .dockerignore generation
- Health check configuration (HEALTHCHECK instruction)
- Docker Compose: services, networks, volumes, secrets
- Local dev compose vs production compose separation
- Port mapping, environment variables, depends-on logic
- Init containers and sidecar patterns
- Resource limits (CPU/memory) configuration
- Nginx / Traefik / Caddy reverse proxy config generation
- Security: non-root user, read-only filesystem, no secret baking
- BuildKit features: parallel builds, layer caching
