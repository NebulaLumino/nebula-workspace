## Research Summary

### LLMOps Landscape
Traditional MLOps handles model training/deployment. LLMOps handles inference: cost, latency, quality, and behavior monitoring.

### Key Tools

**LangSmith** (LangChain)
- End-to-end tracing for LLM apps
- Dataset management, evaluation runs
- Cost per call, latency percentiles
- Most mature LLMOps solution in 2026

**Phoenix** (Arize AI)
- Open source, real-time observability
- Tracing, token counting, prompt versioning
- Embedding drift detection

**Portkey / Helicone**
- AI gateway + logging
- Caching, retry logic, observability

**Datadog LLM Observability**
- APM integration for AI apps
- Correlate AI performance with system health

### Key Metrics to Track
- **Latency**: p50, p95, p99 response times
- **Cost**: per-user, per-model, daily spend
- **Quality**: task completion rate, error rate, user feedback
- **Safety**: toxicity, PII leakage, hallucination rate
- **Drift**: distribution shift in inputs over time

### Project Ideas
- AI cost monitoring dashboard with Vercel Analytics
- Prompt performance tracker with A/B testing framework
