## Research Summary

### What Is AI Agent Safety & Governance?
The field of ensuring AI agents behave correctly, safely, and alignedly — especially when given tools, autonomy, and real-world access. Safety becomes more complex as agents act: they can browse the web, send emails, write code, spend money, make decisions.

### Key Concepts

**Alignment vs. Safety**
- Alignment: making AI do what humans intend
- Safety: preventing AI from causing harm (even if misaligned)
- Agents amplify both: aligned agents do great good, misaligned agents do great harm

**The Agent Threat Model**
1. **Tool misuse**: Agent uses its tools for unintended purposes
2. **Goal drift**: Agent's goals shift during extended operation
3. **Reward hacking**: Agent finds ways to satisfy the letter of the goal but not the spirit
4. **Cascading failures**: Small errors compound into catastrophic outcomes
5. **Overtrust in agent**: Human assumes agent is more reliable than it is
6. **Sandbox escapes**: Agent breaks out of intended execution environment

### Governance Frameworks

**Human-in-the-Loop (HITL)**
- Every significant action requires human approval
- "Human veto" — agent proposes, human decides
- Best for: high-stakes decisions (financial, medical, legal)
- Problem: defeats the purpose of autonomous agents

**Agent Constitution / Behavioral Rules**
- Explicit rules the agent must follow
- Anthropic's approach: Constitutional AI
- Examples: "Do not send emails without human approval for new recipients"
- Layered: primary goal + safety rules + ethical constraints

**Audit Logging**
- Complete record of every action the agent took
- What was called, with what arguments, what was returned
- Timestamps, session context
- Used for: debugging, accountability, compliance

**Scope Limiting / Least Privilege**
- Agent gets only the permissions it needs
- File access: read-only vs. read-write
- API access: only specific endpoints
- Network: allowlist only
- Principle: minimize blast radius of a compromised agent

### Industry Approaches

**Anthropic's Assuredly Safe Autonomy**
- Claude's tool use guidelines
- Explicit capability disclosures
- Constitutional AI for agentic Claude
- Model-specific agent safety training

**Microsoft's AI Agents Governance**
- Azure AI Agent Service with built-in safety
- Content filtering at every tool call
- Resource quotas to prevent runaway agents
- Audit and compliance integrations

**OpenAI's Agent SDK Safety**
- Built-in moderation at every step
- Explicit disclaimers for autonomous actions
- Rate limiting and cost controls
- "Agent can't do X without asking"

### Jailbreaks and Attacks on Agents
- **Prompt injection in tool outputs**: Malicious website injects instructions that agent follows
- **Context window overflow**: Fill context with irrelevant content to cause drift
- **Multi-turn manipulation**: Slowly shift agent's behavior over many turns
- **Tool poisoning**: Compromise a tool the agent depends on
- **Shadow agenting**: A compromised tool acts as an agent without the orchestrator knowing

### Regulatory Landscape
- **EU AI Act**: High-risk AI systems require conformity assessments
- **US Executive Order on AI**: Federal agency guidance on AI safety
- **NIST AI RMF**: Voluntary risk management framework
- **ISO/IEC 42001**: AI management system standard

### Best Practices for Agent Development
1. Start with read-only permissions, add write only when needed
2. Require human confirmation for irreversible actions (delete, send, pay)
3. Implement spending/call quotas
4. Log everything for audit trail
5. Test failure modes explicitly
6. Use sandboxed environments for code execution
7. Add output filtering on every tool result
