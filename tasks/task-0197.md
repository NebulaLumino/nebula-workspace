### How to Evaluate Your AI App

**1. LLM-as-a-Judge** (fast, cheap)
- Use a capable LLM to grade outputs
- Good for: subjective quality, preference
- Risk: LLM has biases

**2. Golden Dataset** (most reliable)
- Human-curated test cases with known correct answers
- Good for: factual Q&A, code generation
- Requires: upfront human effort to build

**3. Synthetic Test Sets**
- Generate test cases with another LLM
- Good for: scaling test coverage
- Risk: biases can compound

**4. Observability Tools**
- Arize Phoenix: traces, token tracking, latency
- DeepEval: pytest-style eval framework
- Ragas: RAG-specific metrics (faithfulness, context precision)
