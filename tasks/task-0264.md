## Research Summary

### What Is Agent Orchestration?
Agent orchestration is the practice of coordinating multiple AI agents to work together on complex tasks — dividing labor, sharing context, and combining their specialized capabilities. A single agent can do one thing; orchestrated agents can tackle multi-step workflows that require different expertise.

### Key Frameworks Explored

**AutoGen (Microsoft)**
- Open-source multi-agent orchestration framework
- Agents can be: AssistantAgent, UserProxyAgent, GroupChatAgent
- Conversations between agents drive workflow
- Code execution built-in (UserProxyAgent runs code)
- Studio UI for visual agent design (AutoGen Studio)
- Strong research backing from Microsoft Research
- Used in: SWE-agent, ChatDev-style dev teams
- Python-first

**CrewAI**
- Role-based multi-agent framework
- Each agent has: role, goal, backstory
- Agents organized into "crews" with sequential or hierarchical processes
- Sequential: Task A → Task B → Task C
- Hierarchical: Manager agent delegates to subordinate agents
- Process-based workflow definition
- Integrates with LangChain tools
- More accessible API than AutoGen

**LangGraph (LangChain)**
- Graph-based orchestration for complex agent workflows
- Nodes = agents or tools; Edges = control flow
- Cycles supported (unlike DAG-based systems)
- Human-in-the-loop checkpoints
- Built-in persistence (memory across runs)
- Good for: complex conditional logic, loops, state machines

**Swarm (OpenAI)**
- Experimental multi-agent orchestration
- Handoffs: one agent transfers to another with context
- Lightweight, simple API
- Not production-ready yet (as of early 2026)
- Good for learning multi-agent patterns

**其他 Notable Solutions**
- **ChatDev**: Virtual software company with agents (CEO, CTO, coder, tester)
- **MetaGPT**: Agents with SOPs — software company simulation
- **CAMEL**: Role-playing agent communication framework

### Orchestration Patterns

**1. Sequential Pipeline**
```
Agent A (research) → Agent B (write) → Agent C (edit)
```
Simple, linear handoff. Good for document generation pipelines.

**2. Hierarchical / Manager**
```
Manager Agent
├── Researcher Agent
├── Writer Agent
└── Reviewer Agent
```
Manager delegates subtasks, synthesizes results.

**3. Parallel / Fan-Out**
```
Task → [Agent A] → Result A
     → [Agent B] → Result B  
     → [Agent C] → Result C
     ↓
Synthesizer → Combined Output
```
Good for: cover multiple aspects simultaneously, then combine.

**4. Debate / Multi-Agent Discussion**
```
Agent A (pro) ↔ Agent B (con) → Arbiter Agent → Decision
```
Good for: complex decisions with competing considerations.

### Communication Patterns
- **Shared message history**: All agents see full context
- **Selective sharing**: Agents share only relevant pieces
- **Tool-mediated**: Agents communicate via shared tools/files
- **Handoff**: Full control transferred to next agent

### Why Orchestration Beats Single Agents
1. **Specialization**: Different agents for different tasks
2. **Quality**: Reviewer catches errors the writer misses
3. **Scale**: Parallel execution across agents
4. **Reliability**: One agent failing doesn't crash whole workflow
5. **Transparency**: Easier to debug individual agents
