## Overview
Build an AI-powered web app that generates Kubernetes manifests and Helm charts from a description of the application deployment needs. Users describe their service (e.g., "a stateless Node.js API with 3 replicas, Postgres database, Redis cache, horizontal pod autoscaling, and TLS termination"), and the AI generates complete Kubernetes YAML manifests and/or Helm chart templates.

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- OpenAI SDK (compatible with DeepSeek)

## Features
- Natural language → Kubernetes manifests (Deployment, Service, ConfigMap, Secret, Ingress, HPA, PVC, StatefulSet)
- Helm chart generation with values.yaml templates
- Kustomize overlay structure (base + overlays/dev/prod)
- Multi-namespace configuration
- Resource requests and limits (CPU/memory)
- Liveness/readiness probe configuration
- Horizontal Pod Autoscaler (HPA) setup
- PodDisruptionBudget (PDB) for high availability
- NetworkPolicy generation (secure by default)
- ServiceAccount and RBAC role/binding generation
- Ingress with TLS, annotation-based config (nginx/istio/gke)
- PersistentVolumeClaim for databases
- Init container and sidecar patterns
- ConfigMap/Secret from key-value pairs or files
- ArgoCD-compatible application manifests
