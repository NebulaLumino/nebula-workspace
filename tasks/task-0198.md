## Description
Research AI-native UX patterns — how to design interfaces that handle AI latency, uncertainty, and streaming responses.

## Focus Areas
- Streaming UI patterns for AI responses
- Handling AI errors gracefully
- Empty states and retry UX
- AI trust and transparency patterns

---

## Research Findings — AI UX Patterns 2026

### Streaming Response UX

**Why streaming matters:** Users feel the AI is "thinking." Even if total time is the same, perceived latency drops. Essential for anything with >1 second response time.

**Best practices:**
- Stream token-by-token — show output as it generates
- Render markdown incrementally (don't wait for full response to display)
- Use a blinking cursor or "typing" indicator during generation
- Debounce token rendering to ~50ms batches (not every single token — too jerky)

### Handling AI Latency

| Latency | UX Pattern |
|---|---|
| <500ms | Normal — no special treatment |
| 500ms-2s | Show "thinking" indicator |
| 2-5s | Progress bar or skeleton UI |
| >5s | Estimated time, cancel button |

### Handling Errors

**AI errors are different from normal errors:** The AI might fail mid-output (partial answer), give wrong info confidently, or time out.

**Best practices:**
- Always show the full error state — never partial answers
- "Retry" should re-run with same prompt (don't lose input)
- For long outputs: checkpoint partial results before running
- Show confidence when possible ("I'm not sure about X — here's my best guess")

### Empty States

| Situation | UX Pattern |
|---|---|
| No input yet | Show example prompts (e.g., "Try: 'Write a birthday poem for my mom'") |
| Generation in progress | Show live output or skeleton loader |
| Generation failed | Clear error + retry button |

### AI Transparency / Trust

- **Show citations** when AI makes factual claims (e.g., in RAG apps)
- **Distinguish AI-generated vs. user content** visually (different backgrounds)
- **Disclose AI involvement** — "This was generated by AI, reviewed by a human"
- **Confidence indicators** — "High confidence" / "This is an approximation"

### Progressive Disclosure
Don't show all options at once. Start with simple UI → expand to advanced options as user grows comfortable.

### For Nova
The streaming text UI pattern is already well-used in Nova's apps. Could improve: add citation display, confidence indicators, and "regenerate with different tone" as an explicit option.
