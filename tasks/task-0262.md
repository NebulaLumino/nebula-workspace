## Research Summary

### What Are Agent Memory Systems?
Agent memory systems give AI agents the ability to retain information across conversations — user preferences, past interactions, learned facts, and session context — enabling truly persistent, personalized AI experiences. Without memory, every conversation starts from scratch.

### Key Platforms Explored

**Mem0 (formerly EmbedChain)**
- Long-term memory layer for AI applications
- Stores user facts, preferences, conversation history
- Semantic search over stored memories
- API-first, easy integration
- Open source option available
- Memory gets "smarter" over time as more interactions occur
- Key concept: "memory entities" — structured facts extracted from conversations

**Letta (founded by MemGPT creators)**
- Stateful LLM agent framework
- Built-in memory management with tiered memory architecture
- Recency weighting, importance scoring
- Multi-agent memory sharing
- Letta Cloud + self-hosted options
- Strong focus on making agents persistent and stateful

**Other Notable Solutions**
- **MemGPT** — Hierarchical memory management for LLMs (tiered: scratchpad, core memory, archival)
- **AutoGen Memory** — Microsoft's approach to agent persistence
- **LangGraph State** — Ephemeral state within LangGraph workflows
- **Narrative Archive** — Document-centric memory for agents

### Memory Architecture Patterns

**1. Hierarchical Memory (MemGPT-style)**
```
Tier 1 (Hot) — Working context: last N messages, active task
Tier 2 (Warm) — Core memory: user profile, preferences, key facts
Tier 3 (Cold) — Archival: full conversation history, old facts
```
LRU eviction from hot → summarization/compression up the tiers.

**2. Entity-Based Memory**
- Extract structured "entities" from conversations
- Entities have: name, type, attributes, last-updated
- Semantic search across entity store
- CRUD operations on entities as agent actions

**3. Vector-Based Memory**
- All conversation chunks embedded and stored in vector DB
- Retrieval: semantic similarity search
- Hybrid: combine vector search + keyword search

**4. Knowledge Graph Memory**
- Facts stored as graph nodes and edges
- Enables reasoning over relationships
- More structured than pure vector

### Key Technical Components
- **Memory extraction** — LLM extracts key facts from conversation
- **Memory retrieval** — Given current context, fetch relevant memories
- **Memory prioritization** — Which memories matter most right now?
- **Memory update** — How to update (merge, overwrite, decay)?
- **Memory forgetting** — When to delete or archive old info

### Why It Matters for AI Products
- Personalization: "remember my preferences"
- Continuity: "continue where we left off"
- Efficiency: "don't ask me the same questions twice"
- Trust: agents that know you feel more intelligent
