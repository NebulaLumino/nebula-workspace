## Research Findings — AI Text-to-SQL and Data Agents 2026

### Leading Tools

**Vanna 2.0** (vanna.ai)
- Personalized AI SQL agent — natural language → SQL
- RAG-based: stores question-SQL pairs as training data → improves with use
- Self-reinforcing: correct SQL saved back to training set → feedback loop
- Agentic architecture with Claude 4.5 + GPT-5 support
- Enterprise security: row-level security, group access, audit logging
- Supports: PostgreSQL, MySQL, Snowflake, BigQuery, Redshift, DuckDB, ClickHouse, and more
- Built-in chat UI component (`<vanna-chat>`)
- NVIDIA NIM integration for accelerated inference

**LangChain Text-to-SQL Agent**
- Schema discovery → LLM generates SQL → validate → execute → error retry → format results
- Supports JOINs, aggregations, subqueries
- Dialect-aware via SQLAlchemy
- LangSmith tracing for debugging
- Key challenge: semantic understanding (LLMs know syntax, not your business terms)

**Other notable tools:**
- **Defog AI**: Conversational analytics
- **SQL Chat**: Chat with your database (via CLI/interface)
- **Querio**: BI platform with natural language
- **Microsoft Copilot**: In Fabric, Azure SQL

### How Text-to-SQL Works (Architecture)

```
User question (NL) → Schema lookup → LLM generates SQL → Validate → Execute → Results → Format → User
                                    ↑
                    (Error? → Rewrite → Retry)
```

**Key challenge:** Syntactically correct ≠ semantically correct
- "Show me sales last quarter" — semantic ambiguity without business context
- Fix: Attach business glossary / semantic layer to prompt

### Multi-Agent Text-to-SQL (2026 trend)
One agent identifies relevant tables, another writes the query, another validates it. Distributes the cognitive load → better accuracy on complex queries.

### Data Agent Patterns

Beyond query generation, **data agents** can:
- Explore a dataset autonomously (find correlations, anomalies)
- Generate charts / visualizations from query results
- Monitor a database and alert on anomalies
- Run scheduled reports and deliver them

### Risks and Mitigations

| Risk | Mitigation |
|------|-----------|
| SQL injection | Validate + sanitize; read-only database user |
| Semantic errors (wrong answer) | RAG over known Q&A pairs; attach schema docs to prompt |
| Expensive queries (full table scan) | Query timeout; cost limits |
| Sensitive data exposure | Row-level security; result filtering |
