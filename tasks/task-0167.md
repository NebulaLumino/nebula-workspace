## Research Findings — RAG Pipeline Architectures 2026

### The Modern RAG Pipeline (Multi-Stage)

```
Query → Query Transformation → Retrieval → Reranking → Generation → Evaluation
```

**1. Data Ingestion**
- Connect to raw sources: PDFs, web pages, wikis, databases
- Extract text + metadata
- Ensure data freshness (incremental indexing)

**2. Chunking & Indexing**
- Beyond character-count splits: semantic chunking, parent-document retrieval
- Multi-representation indexing: embed summaries, return full doc at query time
- Hierarchical indexing: chunk → section → document
- Stored in vector DB (Pinecone, Chroma, Weaviate, Qdrant) + optional traditional DB

**3. Query Transformation**
- **Rewrite:** LLM rephrases with domain-specific language
- **Decompose:** Break complex query into sub-queries → combine results
- **Augment:** Add context to short/vague queries
- **HyDE:** Generate hypothetical document → use it to retrieve

**4. Retrieval**
- Dense (vector semantic) + Sparse (BM25 keyword) = **Hybrid Search**
- Reciprocal Rank Fusion (RRF) combines scores from both
- 20-40% higher recall than dense-only

**5. Reranking**
- Initial retrieve 20-50 candidates → reranker scores against query
- Cross-encoder (BGE, Cohere Rerank 3.5, Voyage Rerank-2.5): jointly encodes query + doc
- Pass top 3-8 to LLM
- Improves precision 18-42%, reduces hallucinations

**6. Generation**
- LLM generates with refined context
- Citations = groundedness signal

**7. Evaluation (Critical!)**
- **Retrieval metrics:** Context Precision, Context Recall, MRR, nDCG
- **Generation metrics:** Faithfulness, Answer Relevancy, Hallucination Rate, Citation Coverage
- **Tools:** Ragas, DeepEval (pytest-style), LangSmith, Arize Phoenix, Maxim AI
- **LLM-as-a-Judge:** Use capable LLMs to grade retrieval + generation quality

### Key Architecture Decisions

| Decision | Options | Trade-offs |
|----------|---------|------------|
| Chunk size | Small (256-512) vs large (1024+) | Smaller = precise; larger = more context |
| Embedding model | OpenAI ada, Cohere, open (BGE) | Quality vs cost vs local |
| Vector DB | Pinecone (cloud), Qdrant, Weaviate, Chroma | Managed vs self-hosted |
| Search type | Dense only vs hybrid | Hybrid best for complex docs |
| Reranking | Yes/no + which model | Precision vs latency/cost |
