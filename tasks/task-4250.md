## Overview
Build an AI-powered web app that generates structured incident postmortems from on-call data. Users paste incident timeline data (from PagerDuty, Datadog, AWS CloudWatch, or raw chat logs), and the AI generates a blameless postmortem document with timeline reconstruction, root cause analysis, impact assessment, action items, and preventive measures.

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- OpenAI SDK (compatible with DeepSeek)

## Features
- Paste incident data: PagerDuty incident JSON, Datadog monitor history, AWS CloudWatch logs, Slack/Opsgenie chat logs, raw text timeline
- AI timeline reconstruction from raw events
- Root cause analysis (fishbone / 5 Whys methodology)
- Impact assessment: users affected, revenue impact, SLA breach duration
- Severity classification: SEV1 / SEV2 / SEV3 / SEV4
- Blameless postmortem sections: Summary, Impact, Timeline, Root Cause, Contributing Factors, Action Items
- Action item generator with owners, priority, and due dates
- "Lessons learned" section in plain language
- Monitoring/alerting gap detection: "we didn't alert on this — here's what to add"
- DORA metrics impact: how this incident affected deployment frequency and MTTR
- Similar incident pattern detection: has this happened before?
- Editable template with AI suggestions as you type
- Export as Markdown, Confluence, Google Docs, or PDF
