## Overview
Build an AI-powered web app that interprets performance profiling reports and flame graphs. Users paste profiling data from Chrome DevTools, Node.js clinic, Python cProfile, Go pprof, Pyroscope, or Datadog, and the AI explains which functions are bottlenecks, why they are slow, and provides specific optimization recommendations with code examples.

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- OpenAI SDK (compatible with DeepSeek)

## Features
- Paste profiling output from: Chrome DevTools CPU profile, Node.js clinic-doctor/clinic-flame, Python cProfile/pprofile, Go pprof, Pyroscope, Datadog APM, New Relic, Scout APM
- Flame graph interpretation (what's on the hot path)
- Function-level analysis: self time vs inclusive time
- Bottleneck identification: N+1 queries, memory allocations, synchronous I/O
- Memory leak pattern detection
- Slow database query analysis (from query logs)
- Network waterfall analysis interpretation
- Optimization recommendations with before/after code examples
- "Good enough" threshold: is this actually a problem at scale?
- Regression comparison: new profile vs baseline (if two profiles pasted)
- Category classification: CPU-bound, I/O-bound, memory-bound, GC pressure
- Async/await call tree visualization explanation
