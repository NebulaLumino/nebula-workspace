## Research Findings — AI Agents in Production 2026

### Case Studies & Patterns

**Klarna (Production since 2024)**
- AI assistant handles 2/3 of customer service chats (2.3M conversations)
- Equal satisfaction vs. human agents
- 25-40% fewer repeat inquiries
- Deployed widely across enterprise — a proof point that AI agents work at scale

**Common Production Patterns**
- Customer service: First-tier support → escalation to humans
- Coding: Autonomous agents (Devin, SWE-agent) for bug fixing, code review
- Research: Multi-step deep research agents with web search + synthesis
- Operations: Automated DevOps workflows (CI/CD, incident response)

### Agent Failure Modes

| Failure Mode | Example | Mitigation |
|---|---|---|
| Tool call errors | Wrong API parameters, missing permissions | HITL for destructive actions |
| Hallucination | Agent "confidently wrong" | Require citations, ground outputs |
| Infinite loops | Agent calling itself | Execution limits, timeout budgets |
| Context overflow | Long convos drain context | Auto-summarize, memory pruning |
| Prompt injection | User tries to manipulate agent | Input sanitization, sandboxing |

### Cost Management
- Token budgets per task — kill agent if cost exceeds threshold
- Model routing: cheap model for simple tasks, capable model for complex
- Token tracking: LangSmith, AgentOps, Helicone for real-time cost monitoring

### Observability Stack
- **LangSmith**: LangChain-native tracing, eval, feedback
- **AgentOps**: OpenAI agent telemetry
- **Helicone**: Simple LLM API logging (open source or cloud)
- Key: trace every agent step, not just final output

### Operational Lessons
- Start with humans-in-the-loop; automate gradually
- Set clear budgets (time, tokens, API calls) per task
- Autonomous ≠ unsupervised — logging is non-negotiable
- Agent sprawl is real: track how many agents you have running
