## Research Summary

### What Is MCP?
The Model Context Protocol (MCP) is an open protocol developed by Anthropic that enables AI models to connect securely to external data sources and tools — databases, APIs, file systems, and services — in a standardized way. Think of it as "USB for AI models": a universal connector that lets any MCP-compatible AI agent interact with any MCP-compatible tool.

### The Problem MCP Solves
Before MCP, every AI agent integration was custom:
- Developer builds custom code to connect agent → Slack
- Developer builds custom code to connect agent → Postgres
- Developer builds custom code to connect agent → GitHub
- Every integration = bespoke, brittle, one-off

MCP standardizes this: one protocol, any data source, reusable across all agents.

### Architecture

```
┌──────────────────────────────────────┐
│           Host (AI Application)       │
│  e.g. Claude Desktop, Cursor, JetBrains │
└──────────────┬───────────────────────┘
               │ MCP Protocol (stdio/HTTP)
┌──────────────▼───────────────────────┐
│         MCP Client (Agent Runtime)    │
└──────────────┬───────────────────────┘
               │
┌──────────────▼───────────────────────┐
│          MCP Server (Tool)            │
│  e.g. GitHub Server, Postgres Server  │
│  or filesystem, Slack, custom tools   │
└──────────────────────────────────────┘
```

**MCP Server**: Exposes tools/resources via the protocol. Can be local (runs on your machine) or remote (SaaS).

**MCP Client**: Runs in the AI application/agent. Manages connections to multiple MCP servers.

**MCP Host**: The AI application the user interacts with (Claude Desktop, Cursor, etc.)

### MCP Capabilities

**1. Tools (Agent Actions)**
```
mcp__github__create_issue(title, body, labels)
mcp__filesystem__read_file(path)
mcp__postgres__query(sql)
```
Standardized tool call interface.

**2. Resources (Data for Context)**
```
mcp__github__repo_info(owner, repo)  → returns context for the agent
mcp__filesystem__list_directory(path)  → agent sees files without running code
```

**3. Prompts (Reusable Templates)**
```
mcp__github__code_review → "Review this PR for security issues"
```
Stored prompt templates that can be invoked.

### Official and Community MCP Servers

**Official (Anthropic maintained)**
- Filesystem (local file access)
- GitHub (issues, PRs, repos)
- Slack (messaging)
- Google Maps (location services)
- Brave Search (web search)

**Notable Community Servers**
- **PostgreSQL**: Query databases directly
- **Puppeteer**: Browser automation
- **AWS**: Cloud resource management
- **Docker**: Container management
- **Memory**: Persistent agent memory
- **Sentry**: Error monitoring
- **Linear**: Project management
- **Notion**: Notes and wikis

### MCP vs. Tool Use (OpenAI Function Calling)
| | MCP | OpenAI Function Calling |
|--|-----|------------------------|
| Scope | Universal (any model) | OpenAI models only |
| Standard | Open protocol | Proprietary |
| Installation | Self-hosted + registry | API-only |
| Tool sharing | Share MCP servers | Share function schemas |
| Real-time | Can access local resources | Cloud only |

### MCP SDKs
- **Python**: `pip install mcp`
- **TypeScript/JS**: `npm install @modelcontextprotocol/sdk`
- **Go**: `go get github.com/modelcontextprotocol/go-sdk`
- **Rust**: `cargo add mcp`

### MCP Registry / Directory
- **Smithery**: smithery.ai — directory of MCP servers with one-click install
- Community-contributed servers for hundreds of tools
- Search by use case, category, or provider

### Security Model
- **Local servers**: Run on your machine, access your local resources
- **Permission prompts**: Claude Desktop prompts before access
- **Scope limiting**: Servers only get access they declare
- **No remote code execution**: Protocol doesn't allow arbitrary code
- **Audit trail**: Agent shows what tools it used

### Real-World Use Cases

**1. Cursor AI + MCP**
- Cursor IDE with MCP servers for GitHub, Postgres, file system
- Agent can read code, push to GitHub, run DB migrations without leaving IDE

**2. Claude Desktop + GitHub MCP**
- "Create a GitHub issue for the bug I just described"
- Agent uses GitHub MCP server to create the issue directly

**3. RAG with Database MCP**
- Agent queries Postgres, uses results as context for LLM
- Live data from any database as agent context

**4. DevOps Agent**
- MCP servers for AWS, Docker, GitHub, Datadog
- Agent manages entire deployment pipeline
