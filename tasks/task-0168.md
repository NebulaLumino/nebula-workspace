## Research Findings — AI Startup Tech Stack 2026

### The Modern AI App Stack (Layer by Layer)

**Frontend + Deployment**
- **Next.js + Vercel**: Standard for AI apps — fast global CDN, serverless functions, easy AI SDK integration
- **Vercel AI SDK**: `generateText()`, `streamText()` — unified LLM API across providers
- **Vercel AI Gateway**: Single endpoint for multiple models (fallback, load balancing)
- Alternative: Railway, Render, AWS Amplify for more control

**LLM Orchestration**
- **LangChain**: Chains, agents, RAG primitives — most widely used
- **LlamaIndex**: Better for query-focused RAG; tighter data connector ecosystem
- **Haystack**: Good for search-heavy pipelines (deepset)
- For simple apps: direct OpenAI SDK calls — orchestration adds complexity

**LLM Models**
- **API-based:** OpenAI (GPT-4o, GPT-5), Anthropic (Claude), Google (Gemini)
- **Open-source:** Qwen, Mistral, Llama (via Together AI, Groq, or self-hosted)
- **Vercel AI SDK pattern:** provider-agnostic → swap models without rewriting

**Vector Database**
- **Pinecone**: Managed, serverless, scales to billions — most popular for startups
- **Qdrant**: Open source, self-hostable, strong performance
- **Weaviate**: Open source, also does full-text search natively
- **pgvector**: PostgreSQL + vector extension — simplest if you already use Postgres
- **Chroma**: Best for local/ prototypes, not production at scale

**Data Layer**
- PostgreSQL / Supabase for structured data + user management
- Redis for caching + session state
- S3 / R2 for document storage (PDFs, images)
- Data pipelines: Airbyte for ingestion, dbt for transformation

**Observability + Evaluation**
- **Arize Phoenix**: Open-source LLM observability — traces, latency, token tracking
- **LangSmith**: LangChain-native tracing and eval (paid)
- **Helicone**: Simple LLM API observability (open source or cloud)
- **Ragas / DeepEval**: RAG-specific evaluation metrics

**Security + Governance**
- **Auth:** Clerk or Auth0 for user management
- **Data isolation:** Row-level security in Postgres
- **API key management:**agi, AWS Secrets Manager
- Decouple data from AI tooling — avoid lock-in

### "AI-Native" Architecture Principles

1. **Composability** — swap models, vector DBs, orchestration without rewriting
2. **Observability first** — log everything; you can't fix what you can't see
3. **Fail gracefully** — model goes down → cached responses or fallback model
4. **Continuous evaluation** — synthetic test sets + LLM-as-judge to catch regressions
5. **Data flywheel** — collect user feedback → improve retrieval + prompts

### Common Startup Stack in 2026

```
Next.js + Vercel          ← Frontend/Deployment
Vercel AI SDK             ← LLM abstraction
LangChain / LlamaIndex    ← Orchestration
Pinecone / Qdrant         ← Vector DB
PostgreSQL + pgvector     ← Structured data + hybrid search
Redis                     ← Caching
Arize Phoenix             ← Observability
Clerk                     ← Auth
Stripe                     ← Payments
```
