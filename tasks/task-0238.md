## Research Findings — RAG Pipeline Architectures 2026

### Basic RAG Flow
Document → Chunk → Embed → Store in Vector DB → Retrieve top-k → Generate answer

### Advanced RAG Patterns

**1. Hybrid Search**
Combine dense (embedding) + sparse (BM25 keyword) retrieval:
```python
results = 0.7 * dense_search(query) + 0.3 * sparse_search(query)
```
Best of both: semantic + keyword matching.

**2. Reranking**
First retrieve 20-50 candidates → then rerank with cross-encoder for precision:
- Cohere Rerank: best-in-class reranking API
- BGE-reranker: open-source alternative

**3. Query Decomposition (Multi-hop)**
Break complex question into sub-questions:
"What did the CEO say about AI at the 2024 earnings call?"
→ Sub-queries: "2024 earnings call", "CEO statements on AI"

**4. Knowledge Graph + Vector Hybrid**
- Entities stored in knowledge graph (structured)
- Text chunks stored in vector DB (unstructured)
- Query both, merge results
